### 项目描述

Mini-Spring 是我独立实现的一个轻量级的 IoC 容器与 AOP 框架，旨在深度剖析和实现 Spring 框架的核心功能，包括控制反转（IoC）容器的设计与实现、面向切面编程（AOP）的代理机制、以及 Bean 生命周期管理等。该项目致力于通过深入实现和优化基础框架，来理解 Spring 容器的工作原理和底层机制。

### 个人工作

+ **IoC容器设计与实现：** 设计并实现了类似 Spring 的 IoC 容器，支持 Bean 的注册、依赖注入和生命周期管理。重点解决了如何在容器中动态管理 Bean 的实例，包括懒加载、单例模式和原型模式的支持。
+ **AOP框架实现：** 完成了 AOP 框架的设计与实现，支持前置通知、后置通知等常见通知类型，利用 JDK 动态代理和 CGLIB 代理技术，灵活实现了目标对象的增强。
+ **Bean 生命周期管理：** 负责设计 Bean 的创建、初始化、销毁等生命周期管理机制，处理 Bean 定义、依赖注入和对象的实例化过程。
+ **依赖管理与接口设计：** 利用 Java 的反射机制实现了 Bean 定义与配置的解析，采用灵活的接口设计确保框架的扩展性和可维护性。
+ **性能优化：** 针对框架实现的性能瓶颈进行了优化，尤其是在 Bean 实例化过程中的延迟加载和缓存机制上，确保框架的高效性。

### 项目难点

+ **AOP 代理实现：** AOP 代理机制中，JDK 动态代理和 CGLIB 代理的选择是一个技术难点。我在实现时深入理解了 JDK 动态代理的接口设计和 CGLIB 代理的字节码操作，成功将这两种代理方式结合，确保了代理的灵活性和性能。
+ **反射与性能优化：** 在通过反射机制创建和注入 Bean 的过程中，性能是一个瓶颈。我通过缓存 BeanDefinition 和工厂方法的反射调用结果来减少反射的使用频率，从而提升了性能。

### 个人收获

+ **深入理解了 Spring IoC 和 AOP：** 通过实现自己的 IoC 容器和 AOP 框架，我对 Spring 容器的核心设计原理、Bean 生命周期的管理、AOP 的代理机制有了更深刻的理解，能够更好地把握企业级框架的设计思想。
+ **反射与性能优化经验：** 在实现过程中，反射机制的使用是关键，但也带来了性能问题。我学会了如何合理使用缓存和优化反射的调用，避免了性能瓶颈。
+ **设计模式的运用：** 在实现过程中，广泛运用了工厂模式、代理模式、单例模式等设计模式，强化了我对设计模式的理解和应用。
